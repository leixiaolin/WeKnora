# 微信公众号与测试号接入配置

本文说明如何把微信公众平台测试号或已认证服务号接入 WeKnora Agent，使用户在公众号中发送文本消息后，收到 WeKnora 生成的回复。

## 1. 支持范围

WeKnora 中该渠道的平台标识为 `wechat_mp`，与“微信个人号”渠道不同。

当前版本支持：

- 微信公众平台接口测试号。
- 已认证微信服务号。
- 明文模式 Webhook 回调。
- 用户发送文本消息，WeKnora 异步返回纯文本答案。
- 复用 WeKnora 的 Agent、RAG、会话、限流和消息去重能力。

当前版本暂不支持：

- 订阅号主动回复能力保证；建议使用测试号或已认证服务号。
- 安全模式、兼容模式和 `EncodingAESKey` 消息解密。
- 图片、语音、视频、位置、链接和文件消息处理。
- 流式更新同一条微信消息。
- 自定义菜单、模板消息、订阅通知和多客服转接。

微信客服消息存在用户交互窗口限制。用户长时间未与公众号互动时，微信可能拒绝 WeKnora 的异步回复。

## 2. 接入原理

消息链路如下：

```text
微信用户发送文本
  -> 微信服务器调用 WeKnora Webhook
  -> WeKnora 验证签名并解析 XML
  -> WeKnora 立即返回 HTTP 200 空响应
  -> Agent / RAG 在后台生成答案
  -> WeKnora 调用微信客服消息 API
  -> 用户收到纯文本回复
```

回调地址格式为：

```text
https://你的公网域名/api/v1/im/callback/{channel_id}
```

其中 `{channel_id}` 是 WeKnora 创建 IM 渠道后生成的渠道 ID，不能填写 Agent ID、知识库 ID 或 AppID。

## 3. 准备工作

开始前需要准备：

1. 已能正常使用的 WeKnora，并已创建 Agent。
2. 微信公众平台测试号，或已认证微信服务号。
3. 测试号/服务号的 `AppID` 和 `AppSecret`。
4. 微信服务器能够访问的公网 HTTPS 回调地址。
5. 自定义的随机 Token，用于验证微信回调签名。

安全要求：

- 不要在聊天、截图、日志或 Git 仓库中公开 `AppSecret`。
- 如果 `AppSecret` 已泄露，应立即在微信公众平台重置。
- Token 建议使用至少 32 位随机字母和数字，不要与 AppSecret、管理员密码相同。
- 正式部署应使用固定 HTTPS 域名；临时隧道仅用于联调。

## 4. 获取微信测试号信息

如使用接口测试号：

1. 登录微信公众平台，进入“公众平台测试账号”页面。
2. 使用微信扫码登录。
3. 在“测试号信息”区域记录 `appID` 和 `appsecret`。
4. 在“测试号二维码”区域扫码关注测试号。

如使用正式服务号，请在微信公众平台后台的开发设置中获取 AppID，并生成或重置 AppSecret。服务号应完成微信要求的认证，并具备客服消息接口权限。

## 5. 准备公网 HTTPS 地址

### 5.1 已有公网域名

把域名通过 Nginx、Caddy 或其他反向代理转发到 WeKnora 后端。必须确保以下路径能够到达 WeKnora API 服务：

```text
/api/v1/im/callback/
```

例如：

```text
https://weknora.example.com/api/v1/im/callback/{channel_id}
```

### 5.2 本地使用 ngrok 联调

如果本机没有公网 IP，可以把本地后端端口临时映射到公网。确认 WeKnora 后端监听 `8080` 后执行：

```powershell
ngrok http 8080
```

ngrok 会输出类似：

```text
Forwarding  https://example.ngrok-free.app -> http://localhost:8080
```

此时回调地址为：

```text
https://example.ngrok-free.app/api/v1/im/callback/{channel_id}
```

注意：

- ngrok 进程必须保持运行。
- 免费临时域名变化后，必须同步修改微信后台的 URL。
- 隧道应指向后端 API 端口 `8080`，不是前端开发端口 `5173`。
- 可访问 `http://127.0.0.1:4040` 查看 ngrok 收到的验证和消息请求。

## 6. 在 WeKnora 创建微信渠道

1. 登录 WeKnora。
2. 打开要接入的 Agent 编辑器。
3. 进入“IM 集成”。
4. 点击“添加渠道”。
5. 按下表填写配置。

| 字段         | 填写内容                 | 说明                                  |
| ------------ | ------------------------ | ------------------------------------- |
| 渠道名称     | 例如“微信公众号测试号” | 仅用于 WeKnora 内部识别               |
| 平台         | 微信服务号               | 测试号也选择该项，不要选择“微信”    |
| 接入模式     | Webhook                  | `wechat_mp` 固定值                  |
| 输出模式     | 完整输出                 | 微信不支持编辑同一消息模拟流式输出    |
| App ID       | 微信后台的 AppID         | 通常以`wx` 开头                     |
| App Secret   | 微信后台的 AppSecret     | 敏感凭证，不应公开                    |
| Token        | 自定义随机字符串         | 必须与微信后台 Token 完全一致         |
| API Base URL | 留空                     | 默认使用`https://api.weixin.qq.com` |
| 启用状态     | 启用                     | 未启用的渠道不会处理回调              |

保存后，编辑或查看渠道卡片，复制 WeKnora 展示的完整回调地址。确认地址末尾包含真实的渠道 ID，例如：

```text
https://example.ngrok-free.app/api/v1/im/callback/2f43a000-0000-0000-0000-000000000000
```

不要把示例 ID 原样复制到微信后台。

## 7. 配置微信服务器地址

在微信测试号页面找到“接口配置信息”；正式服务号则进入微信公众平台的“服务器配置”。填写：

| 微信字段       | 配置值                         |
| -------------- | ------------------------------ |
| URL            | WeKnora 的完整公网回调地址     |
| Token          | 与 WeKnora 渠道 Token 完全一致 |
| EncodingAESKey | 留空或不配置                   |
| 消息加解密方式 | 明文模式                       |

点击“提交”或“保存”后，微信会向回调地址发送带有 `signature`、`timestamp`、`nonce` 和 `echostr` 的 GET 请求。WeKnora 校验签名成功后会原样返回 `echostr`，微信页面应提示配置成功。

如果使用 ngrok，可以在检查面板中确认请求：

```text
http://127.0.0.1:4040
```

成功验证请求通常具有以下特征：

- 请求方法为 `GET`。
- 路径为 `/api/v1/im/callback/{channel_id}`。
- Query 中包含 `signature`、`timestamp`、`nonce`、`echostr`。
- HTTP 状态为 `200`。
- 响应正文是微信发送的 `echostr`。

## 8. 发送消息验证

1. 用微信扫码关注测试号或服务号。
2. 向公众号发送一条简单文本，例如“你好”。
3. 等待 WeKnora Agent 返回答案。

正常情况下：

- 微信向回调地址发送 XML 格式的 POST 请求。
- WeKnora 很快返回 HTTP `200`，响应正文为空。
- Agent 在后台执行问答流程。
- 数秒后，用户收到客服消息形式的纯文本回复。

回答中的 Markdown 会被转换成微信纯文本；较长答案会被自动分片。微信重复投递相同 `MsgId` 时，WeKnora 会按渠道和消息 ID 去重，避免重复执行问答。

## 9. 常见问题排查

### 9.1 微信提示“配置失败”

依次检查：

1. URL 是否为公网可访问的 `https://` 地址。
2. URL 是否包含 `/api/v1/im/callback/{真实渠道ID}`。
3. ngrok 或其他隧道是否仍在运行。
4. 隧道是否指向 WeKnora 后端端口，而不是前端端口。
5. 微信和 WeKnora 中的 Token 是否逐字符一致，注意大小写和前后空格。
6. WeKnora 渠道是否已启用。
7. 反向代理是否保留 Query 参数，并把该路径转发到后端。
8. 后端日志中是否出现渠道不存在、签名错误或时间戳过期。

不要用浏览器直接打开回调 URL 来判断是否正常。缺少微信签名参数的普通请求被拒绝是预期行为。

### 9.2 微信配置成功，但发消息没有回复

检查：

1. 微信是否向同一个回调 URL 发出了 POST 请求。
2. POST 请求是否返回 `200` 且响应正文为空。
3. Agent 是否已正确配置模型和知识库，且在 Web UI 中能够正常回答。
4. AppID 和 AppSecret 是否属于同一个测试号或服务号。
5. AppSecret 是否已重置但 WeKnora 中仍保存旧值。
6. 微信客服消息接口是否有权限，用户是否仍处于客服消息交互窗口内。
7. 后端是否能访问 `https://api.weixin.qq.com`。

### 9.3 回调出现 401、403 或签名错误

- 重新复制 Token，确保微信和 WeKnora 完全一致。
- 检查系统时间是否准确；WeKnora 会拒绝超过允许时间窗口的回调，以降低重放风险。
- 不要让代理删除或改写 `signature`、`timestamp`、`nonce` 查询参数。
- 确认没有把微信个人号渠道的回调地址用于服务号。

### 9.4 微信反复投递同一条消息

微信通常会在回调超时或响应不符合预期时重试。检查 WeKnora 是否在 5 秒内返回：

```text
HTTP 200
空响应正文
```

不要让网关把空响应改写成 JSON、HTML 错误页或统一响应结构。

### 9.5 能收到短回答，长回答不完整

WeKnora 会把长文本拆成多条客服消息发送。若部分分片失败，检查微信接口限流日志。达到调用频率限制时，WeKnora 不会立即反复重试，以免进一步放大限流。

### 9.6 错误码 `45015`

该错误通常表示用户已不在客服消息允许的交互窗口内。让用户重新向公众号发送一条消息后再测试。WeKnora 会记录警告，但不会无限重试。

### 9.7 错误码 `40001` 或 `42001`

表示 access token 无效或过期。WeKnora 会清除 token 缓存并重试一次；如果仍失败，请检查 AppID、AppSecret，以及微信后台是否刚刚重置过密钥。

## 10. 生产环境建议

- 使用固定域名和有效 HTTPS 证书，不使用临时 ngrok 域名长期运行。
- 仅暴露必要的 Webhook 路径，通过反向代理转发到 WeKnora。
- 保持服务器系统时间同步。
- 定期轮换 AppSecret，并通过 WeKnora UI 或安全配置渠道更新。
- 不在日志中打印 AppSecret、access token、Token 或完整敏感 Query。
- 自定义 API Base URL 时只使用可信 HTTPS 地址，并确保符合 SSRF 防护策略。
- 监控回调状态码、处理耗时、微信接口错误码和客服消息限流。
- 如果答案包含对象存储临时 URL，确认该 URL 可由用户微信客户端访问且不会泄露跨租户资源。

## 11. 配置检查清单

接入完成后逐项确认：

- [ ] 使用的是测试号或已认证服务号。
- [ ] 已安全保存 AppID 和 AppSecret，凭证未提交到 Git。
- [ ] WeKnora 渠道平台选择“微信服务号”。
- [ ] 接入模式为 Webhook，输出模式为完整输出。
- [ ] 微信与 WeKnora 的 Token 完全一致。
- [ ] 回调 URL 使用公网 HTTPS，并包含真实 `channel_id`。
- [ ] 微信服务器配置选择明文模式。
- [ ] 微信 GET 接入验证成功。
- [ ] 用户发送文本后，回调 POST 返回 `200` 空正文。
- [ ] 用户能收到 WeKnora 的客服消息回复。
- [ ] 日志中没有 AppSecret、access token 等敏感信息。
